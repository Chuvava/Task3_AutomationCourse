  ОПИСАНИЕ ЗАДАНИЯ:
Разработать автоматический тест c возможностями запуска на Firefox и Google Chrome по следующему сценарию:
     1) Переход на http://store.steampowered.com/
     2) В меню Games(Игры) выбираем Actions(Экшены):
     3) Переходим на вкладку Specials(Скидки)
     4) Выбираем игру из условия максимальной скидки(только на 1-ой странице, навигация не требуется)
            4.1) В случае появления вопроса о возрасте – заполняем валидными данными и жмем Enter
     5) На странице с описанием игры проверяем корректность цены и скидки. За эталон взять те цифры, что на шаге 4.
     6) Переходим по ссылке Install Steam в верхнем правом углу.
     7) Качаем setup файл (шаг подразумевает ожидание окончания загрузки файла)

  КРИТЕРИИ УСПЕШНОГО ВЫПОЛНЕНИЯ:
1. Созданный проект должен автоматически собираться без дополнительного вмешательства человека
(например не для запуска тестов не должно быть необходимо что-то качать(к примеру библиотеки) и т.п. - всё должно быть в проекте или выкачиваться автоматически)
Простыми словами - чтобы запустить тесты нужно - скачать проект, открыть в IDE и запустить

2. Проект должен быть организован и запускаться через UnitTest Framework
3. Тест должен работать в 2х браузерах (Firefox, Chrome), при этом тип браузера настраивается в конфигурационном файле
4. В задании должны быть использованы в основном XPath локаторы
5. Структура приложения должна быть построена на основании принципов делегирования и распределения реализации работы с функциональностью в зависимости от принадлежности к определенному компоненту графического интерфейса
Например есть Form и есть Menu - на Form может быть экземпляр Menu так, чтобы из теста мы могли обратиться к меню следующим образом:
form.menu.selectItem(itemName)
6. Взаимодействие с приложением должно быть организовано посредством элементов наследников от Element. При необходимости нужно добавлять своих наследников.
7. В классах тестов и форм не должно быть обращения к методам findElement(s) напрямую
8. Тест должен проверять что загруженный файл появился в файловой системе (windows и linux)
9. Код должен соответствовать Code Conventions (для проверки можно установить триальную версию Resharper для C# или CheckStyle/PMD для Java). Сами Code Conventions можно найти в интернете по запросу: code conventions for [LANGUAGE]
10. Тест должен запускать как на англо так и на рускоязычной версии сайта.
